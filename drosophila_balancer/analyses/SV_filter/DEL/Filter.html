Filter by read support

First I would argue that an extremely high ref or alt support means there is something wrong, so I remove them:

bad_ids = unique( subset(d, (precise == "precise" & (RR >= 500 | RV >= 500) )
                          | (precise == "imprecise" & (DR >= 750 | DV >= 750) ))$id )
d = subset(d, !(id %in% bad_ids))

Then I will require a minimum support. For precise calls, require at least 25 total reads in the cross and 15 total reads in the virginizer. For imprecise, let's say 40 and 20. This cutoff is shown as a dashed orange line in the plots below.

Filtering by read depth signal

It looks like for “large” deletions the signal is actually useful. We can apply a filter to that. Let's take the 10% - 90% quantiles for example:

RClim = subset(d,big=="big") %>% 
        group_by(GT, sample) %>% 
        summarise(y = median(RC/(RCL+RCR)), 
                  ymax = quantile(RC/(RCL+RCR), 0.9), 
                  ymin = quantile(RC/(RCL+RCR), 0.1) )

Filter by read depth ratio between samples

The only way read depth filters make sense is in a comparison between samples because the read depth signal is highly influenced by mappability and random fluctuations in coverage.

d$RC_ratio = d$RC / (d$RCL + d$RCR)
RDratios = dcast(subset(d, big=="big"), id + size + combinedGT + precise ~ sample, value.var="RC_ratio")

Let's first look at the genotypes 0/1 0/0. Here we would expect inside-to-outside read depth ratios of 50% in the cross and 100% in the virginizer. But since this is not relevant due to mappability issues, we'll only look at the difference of these ratios and require a minimum difference, i.e. that \(r_{vrg} \geq r_{cross} + 0.2\).

Actually, we expect random fluctuations on intervals to decrease with interval size. This is why a size-dependent or dynamic cut-off makes much more sense.

I want to investigate this dependncy more in the future. This might be a useful filter for Delly

Next, for genotype 0/1 1/1 we would expect ratios of 50% and 0%, respectively, so looking at the difference, we again require a minimum distance, i.e \(r_{vrg} \leq r_{cross} -0.2\). Also here we will use the dynamic cut-off.

Apply dynamic read depth filter

I manually inspected the calls above 2 kb that get kicked out by the read depth ratio filter and it looks very convincing. As a side note, we are going to put the common deletions into a separate file as they don't allow a comparison between samples.

good_IDs1 = subset(RDratios,  # Fixed Threshold
      combinedGT=="0/1.0/0" & cross < vrg -0.2 |
      combinedGT=="0/1.1/1" & vrg < cross -0.2)$id
RDratios$dynThr = dynThrF(RDratios$size) # Arbitrary function that shrinks in a nice shape
good_IDs2 = subset(RDratios, # Dynamic Threshold
      combinedGT=="0/1.0/0" & vrg-cross < dynThr+0.5 & vrg-cross > 0.5-dynThr |
      combinedGT=="0/1.1/1" & vrg-cross < dynThr-0.5 & vrg-cross > -0.5 -dynThr)$id
d = subset(d, big=="small" | combinedGT == "1/1.1/1" | id %in% good_IDs2)

Stratify by mappability

Especially the common deletions, but also sample-specific ones can be false positives due to a low mappability region. This is why we would like to split the calls into a high-confidence, where at least 80% of the deletion overlap unique regions in the genome, and a low-confidence set.

# find overlaps between deletions and unique regions
uniq = reads <- import.bed("uniqueAreas.bed")#, genome="dm6")
dels = makeGRangesFromDataFrame(subset(d, sample=="cross", select=c("chr","start","end","id")),
                                ignore.strand=T, keep.extra.columns=T)
seqlevels(dels) = seqlevels(uniq)
hits = findOverlaps(uniq, dels)
hits = pintersect(dels[subjectHits(hits)], uniq[queryHits(hits)])

# Select deletions covered to 50% by unique regions
highConf = data.frame(id = hits$id, size = end(hits) - start(hits)+1) %>% 
  group_by(id) %>% 
  dplyr::summarize(total_size = sum(size), intervals = length(size)) %>% 
  inner_join(subset(d, sample=="cross", select=c(id,size)), by="id") %>%
  filter(total_size >= 0.5*size)
d$confidence = factor(d$id %in% highConf$id, levels=c(T,F), labels=c("high","low"))
